Java Strings

  • The Java String class encapsulates a sequence of unicode (UTF-16) characters
  • Strings are immutable, meaning that once a string instance is created cannot be modified. The String object itself is effectively constant
  • Strings can be created multiple ways: 
    
final String s1 = "Cooper"; // the new operator is not required for String
final String s2 = new String("Annie");
final String s3 = """
                  Brandy"""; // textblock
  • String concatenation is performed using the binary + operator
  • String appending (which uses concatenation) can be performed using the += operator
  • Concatenation rules:
    1. For two numeric operands, + operator performs addition
    2. If either operands is a String, then + performs String concatenation
    3. The associativity for the binary + operator is always left-to-right
    
System.out.println("green" + "-fish"); // outputs green-fish
System.out.println(1 + 2 + "ab"); // outputs 3ab

final int three = 3;
final String four = "4";
System.out.println(1 + 2 + three + four); // outputs 64

var s = "1";
s += "2";
s += 3;
System.out.println(s); // outputs 123
System.out.println(s + null); // outputs 123null

Equality of Variables

  • The binary == operator compares object references (when used with references) to check if they are pointing at the same object
  • The equals(Object obj) method should be overriden to compare the member fields of a class
  • The String Pool
    • Java stores strings in the string pool (also known as the intern pool) which is a location in the JVM stores all strings
    • Repeated strings are reused to reduce memory usage
    • The method public String intern() tells java to retrieve the String from the string pool, or add it if it's not present.
    • The two String constructors indicate whether to use the string pool: 
      
var x = "Hello World";   // uses the string pool
var y = new String("Hello World");  // DOES NOT use the string pool, no reuse
var z = new String("Hello World").intern() // uses the string pool if string is available
System.out.println(x == "Hello World"); // outputs true
System.out.println(x == y); // outputs false
System.out.println(x == z); // outputs true
System.out.println(x == y.intern()); // outputs true

Java Arrays

  • Arrays are an ordered list object, which can be created from any Java type. The unary operator [] is the array access operator, which returns the variable type at the specified index. An array initializer can be specified using braces {}. 
    • 
final int[] numbers = new int[3]; // the type of reference numbers is int[]
final int[] moreNums = new int[] {10, -5, 2, 10_020};
final int[] nums3 = {-2 100 2 5 -2 5};  // known as an Anonymous Array
  • If the array initializer is not specified, a newly constructed array has its elements initialized to its type's default value.
  • When specifying the array type, the square brackets can be written after the type or after the identifier, with or without a space(s).
    • 
// these all compile
int[] numAnimals;
int [] numAnimals2;
int []numAnimals3;
int numAnimals4[];
int numAnimals5 [];
  • Multiple variable declaration:
    • int[] x, y; declares two array references of type int[] as expected.
    • *Note int x[], y; declares one array reference int[] x and one int y.
  • The equals(Object obj) method for arrays only looks for reference equality, it does not inspect array elements if the references are not equal
  • Array Properties and Utilities
    • The array member arr.length returns the length (number of elements)of the array.
    • The unary array access operator arr[index] is used to access and return the element at the specified index of an array. Trying to access an index equal or greater than the array length throws an ArrayIndexOutOfBoundsException.
    • Looping through an array:

    • Searching a presorted array: Arrays.binarySearch(Arrays.sort(numbers));

    • public static boolean Arrays::equals(T[] a, T[] b); - returns true if all elements in the arrays are equal
    • public static int Arrays::compare(T[] a, T[] b);
      • Return value:
        • Negative number: the 1st array contains a lesser value for the first element that differs, or is shorter than the 2nd
        • A zero mean the arrays length and elements are all equal
        • Positive number means 1st array contains a greater value for the first element that differs, or is longer
      • Other compareTo() rules:
        • null is smaller than any other value
        • For strings: a prefix is smaller than the parent
        • For strings: a number is smaller than a letter
        • For strings: an uppercase letter is smaller than a lowercase letter
    • public static int Arrays::mismatch(T[] a, T[] b) - returns the index of the first mismatched/differeing element of the 2 arrays. Returns -1 if the arrays are equal.
    • public static String toString(T[] a) - returns a String representation of the array, usually printing out the String representation of each element

Math APIs

  • java.lang.Math contains many static method for performing mathematical calculations, and does not require an import statement.
  • public static double Math::min(double a, double b); - compute minimum of 2 values, override methods also exist for float, int, and long
  • public static double Math::max(double a, double b); - compute maximum of 2 values, same overrides as min
  • public static long Math::round(double num); - rounds off a double value with 0.5 rounding up
    public static int Math:round(flat num); - rounds a foat value
  • public static long Math::ceil(double num); - rounds up to nearest integer, no override for other parameter datatypes.
    public static long Math::floor(double num); - rounds down to nearest integer, no overrides either
  • public static double Math::pow(double number, double exponent) - computes the power of the specified number, and exponent can be a fraction.
  • public static double Math::random() - generates a random double value in interval: [0, 1)

Dates and Times

  • Java's modern date and time API is: import java.time.*;
  • Date and time classes that store an immutable date, time, or date and time. Note that months and days start from 1, while hours, minutes and seconds start at 0, respectively.
    • LocalDate - stores a specific day int time, but not any time (less than 24hours) information

      • final var t2 = LocalDate.of(int year, int month, int dayOfMonth);
      • final var t3 = LocalDate.of(int year, Month month, int dayOfMonth);
    • LocalTime - stores an time instant, but no day/date information
      • final var t1 = LocalTime.now();
      • final var t2 = LocalTime.of(int hour, int minute);
      • final var t3 = LocalTime.of(int hour, int minute, int second);
      • final var t4 = LocalTime.of(int hour, int minute, int second, int nanos);
    • LocalDateTime - stores the local date and time
